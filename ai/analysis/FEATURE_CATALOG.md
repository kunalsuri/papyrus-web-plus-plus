## §2 API / Interface Surface  `[inferred]`

### GraphQL (Papyrus-specific — the complete set defined in this repo: 30 operations)

Repo-wide annotation scan across all `backend/*/src/main/java` trees: **30 data
fetchers** — 11 core ones (profiles/stereotypes/metaclasses, first table) plus 19
widget-scoped ones (second table) in `backend/papyrus-web-custom-widgets/`.
Everything else on the GraphQL API is Sirius Web's (external). Papyrus's schema
extensions are declared in 7 `.graphqls` files:
`backend/papyrus-web-application/src/main/resources/schema/profile.graphqls`,
`backend/papyrus-web-application/src/main/resources/schema/customnodes.graphqls`,
and one per widget under `backend/papyrus-web-custom-widgets/src/main/resources/schema/`.

#### Core (11 operations) — in `backend/papyrus-web-application/src/main/java/org/eclipse/papyrus/web/application/profile/controllers/` and `backend/papyrus-web-application/src/main/java/org/eclipse/papyrus/web/application/representations/controllers/`

| Operation | GraphQL coordinates | Handler class | Feature |
|---|---|---|---|
| `profileMetadatas` | Query `Viewer.profileMetadatas` | UMLProfileMetadatasDataFetcher | F1 |
| `profileLastVersion` | Query `EditingContext.profileLastVersion` | EditingContextProfileLastVersionDataFetcher | F1 |
| `currentEditingContext` | Query `UMLProfileMetadata.currentEditingContext` | ProfileCurrentEditingContextDataFetcher | F1 |
| `stereotypeMetatadas` (sic — upstream typo) | Query `EditingContext.stereotypeMetatadas` | UMLEditingContextStereotypesDataFetcher | F2 |
| `publishProfile` | Mutation | MutationPublishProfileDataFetcher | F1 |
| `deletePublishedDynamicProfileByName` | Mutation | MutationDeletePublishedDynamicProfileDataFetcher | F1 |
| `applyProfile` | Mutation | MutationApplyProfileDataFetcher | F2 |
| `applyStereotype` | Mutation | MutationApplyStereotypeDataFetcher | F2 |
| `metaclassMetadatas` | Query `EditingContext.metaclassMetadatas` | EditingContextMetaclassesDataFetcher | F3 |
| `createMetaclassImport` | Mutation | MutationCreateMetaclassImportDataFetcher | F3 |
| `isProfileDiagram` | Query `RepresentationMetadata.isProfileDiagram` | RepresentationMetadataIsProfileDiagramDataFetcher | F3 |

Each mutation is executed by a matching `IEditingContextEventHandler` in
`backend/papyrus-web-application/src/main/java/org/eclipse/papyrus/web/application/profile/handlers/`
(6 handlers) and `backend/papyrus-web-application/src/main/java/org/eclipse/papyrus/web/application/representations/handlers/`
(2 handlers).

#### Widget sub-API (19 operations) — in `backend/papyrus-web-custom-widgets/` (F7)

`.graphqls` schema file in `backend/papyrus-web-custom-widgets/src/main/resources/schema/`.

| Containment reference | `ContainmentReferenceWidget.containmentReference`, `ContainmentReferenceItem.iconURL`, `ContainmentReferenceItem.hasClickAction` | `clickContainmentReferenceItem`, `moveContainmentReferenceItem`, `removeContainmentReferenceItem` |

| Language expression | — | `addLanguage`, `deleteLanguage`, `moveLanguage`, `editLanguageBody` |
| Custom image | — | `newUuid`, `removeUuid` |
| Primitive radio | — | `newValue` |

## §4 Where New Code Lives — decision tree  `[inferred]`

```
What kind of change?
├── Touches an inherited feature (F1–F16)?
│   └── STOP — that code is frozen (fork rule). Read it via this catalog;
│       edit only if the task EXPLICITLY requires changing upstream code.
├── New fork-side capability?
│   ├── Dev workflow / build / run scripts      ➔ scripts-pwpp/
│   ├── Fork documentation                      ➔ doc-pwpp/
│   ├── Agent knowledge & configs               ➔ ai/  (+ .claude/ .agents/ .cursor/)
│   └── New runtime feature code                ➔ a NEW module (new Maven module or npm
```
