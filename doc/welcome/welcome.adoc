== What is Papyrus-Web?[[WhatIsPapyrus-Web]]
*Papyrus-Web* is an open source software dedicated to the *UML* diagrams modeling. *Papyrus-Web* is the modern version of *Papyrus-Desktop*, another UML modeling tools Eclipse based. 
*Papyrus-Web* is built on the top of *Sirius-Web*, a generic modeling tool. We use a Client-Server architecture, with a PostGreSQL database to save the models. This architecture allows to centralize the models and avoid the installation to the final user. Due to its construction, *Papyrus-Web* fully support the Collaborative Work.

The purpose of the *Papyrus* project is to provide a UML editors fully compliant with the UML standard defined by the https://www.omg.org/[OMG] (Object Management Group).

*Papyrus-Web* provides to the final user the main UML Diagrams, with three views to edit its models:
 
* the Explorer,  on the left. This view allows to navigate in the model;
* the editor view, on the center. This place is used to edit the diagrams, the tables and some other views;
* the Details view, on the right. This view allows to edit the properties of the selected element.

image::classDiagramExample.png[title="Example of Model, with a ClassDiagram", align="left"]

*Papyrus-Web* has be designed to be reusable and extensible (under the criteria covered by the https://www.eclipse.org/legal/epl-2.0/[license EPL v2]). So, first at all, *Papyrus-Web* provides the *UML Profile diagram* which allows to specialize the concepts provided by UML to adapt them to specific domain. In addition, the final user can customize the exiting diagrams and create new ones for its domain. In addition, the final user can connect its own tools (code generator, validator, ...) to target its specific needs.


== What is UML? 
*UML* means Unified Modeling Language. It is a standard defined by the Object Management Group (https://www.omg.org/spec/UML/[UML Norm]).
*UML* is a graphical language providing around than 250 concepts (like Class, Package, Lifeline, Usecase, ...) and defining 14 diagrams. There are three kinds of diagrams:

=== Structure Diagrams

* Class Diagram, 
* Component Diagram, 
* Composite Structure Diagram,
* Deployment Diagram, 
* Object Diagram, 
* Package Diagram, 
* Profile Diagram.

=== Behavior Diagrams

* Activity Diagram,
* StateMachine Diagram, 
* Use case Diagram.

=== Interaction Diagrams

* Communication Diagram,
* Interaction Overview Diagram,
* Sequence Diagram,
* Timing Diagram.

*UML* provides an extension mechanism called ``Profile``. This mechanism allows to specialize the generic UML language to the requirements of each domain. This capabilities is fully supported by *Papyrus-Web*
